TECHNIQUES DE L'INFORMATION ET DE LA COMMUNICATION

Chapitre 1: introduction

  1. Les techniques de l'information et de la communication (TIC):

    2. Les TIC sont des téchnologies ou des outils pour collecter, traiter,
    stocker,transmettre et diffuser des informations via différents
    médias et canaux de communication.

  2. Catégories et objectifs :


    • a. Les catégories :
      1. La téléinformatique .
      2. Télécommunication.
      3. L'électronique.
    • b.Les objectifs recherchés :
      1. La connectivité .
      2. La modularité .
      3. Une implantation simple.
      4. Une utilisationfacile.
      5. La fiabilité.

  3. Composants de TIC:


    • a. La partie matérielle:
      1. Pc portable, la télévision, le téléphone portable...
      2. Station de travail : toute machine capable denvoyer
        des données vers les réseaux .
    • b.La partie logicielle:
      1. Les systèmes d'éxploitation, les logiciels d'application
        et d'utilisation desdifférents composants matériels.
      2. Les réseaux d'internet ou mobiles.

  4. Caractéristiques:


    • a- La rapidité .
    • b- La décentralisation du pouvoir.
    • c- Un meilleur accès aux informations .
    • d- La rationalité .
    • e- La mondialisation .
    • f- La répartition .

  5. Initiation aux techniques de communication:


    • Objectifs de communication persuader et
      Modifier ou renforcer limage de produit, générer des ventes....

      1-Les vecteurs : message, media.

      3-Bruit.

    • Les types de communication :

      1. La communication institutionnelle.
      2. La communication commerciale.

    • Les schémas de communication :
      1. La communication interpersonnelle : un émetteur va transmettre

      1. Média : est un moyen de communication utilisé pour diffuser

    2. Ensemble d’ordinateurs et des périphériques libres
    interconnectés pour la communication de données entre
    applications quelles que soient les distances qui les séparent.
    (Remarque : un réseau est un connexion de deux ordinateurs en moins.)

      • Les réseaux informatiques ont été créés pour permettre la
      communication entre des terminaux distants et un site central
      , puis entre ordinateurs. Au départ, ces réseaux étaient
      utilisés uniquement pour le transport de données informatiques
      , mais maintenant ils intègrent également la parole et la vidéo.

    • Networking :


      • Mise en œuvre des outils et des tâches permettant de
      relier des ordinateurs pour le partage des ressources.

Chapitre 3: le Model OSI

  1. Les Composants matériel d’un Réseau informatique :

    2. Les cartes réseau:

    ou NIC, sont les composants qui
    permettent à l’ordinateur de se connecter à un réseau via une
    connexion Ethernet ou Wi-Fi. Elles facilitent la transmission
    et la réception de données via des protocoles de communication tels que TCP/IP.
    Les switches : sont les composants utilisés pour
    connecter les appareils sur un réseau local (LAN), en stockons leur
    adresses MAC dans une table de correspondance pour envoyer les
    données uniquement à l'appareil destinataire. Ils sont utilisés
    dans des réseaux de différentes tailles. Ils offrent également des
    fonctionnalités avancées telles que la gestion des VLAN.

    Les hubs:

    Sont les composants
    utilisés pour connecter plusieurs appareils sur un réseau local
    et transmettre les données à tous les appareils connectés.
    Bien que les hubs
    aient été remplacés par des switches plus
    rapides et plus sécurisés, ils sont toujours utilisés dans
    les petits réseaux.

  2. Le Modèle OSI :


    3. Le modèle OSI est un modèle de référence qui définit comment
    les systèmes réseau communiquent et envoient des données d'un
    expéditeur à un destinataire. Il est divisé en sept couches,
    (la couche physique, la couche liaison de données, la couche
    réseau, la couche transport, la couche session, la couche
    présentation et la couche application) chacune ayant des
    fonctions spécifiques. Bien qu'il ne soit pas utilisé dans
    les protocoles de communication modernes, il reste toujours
    utile pour comprendre les principes de la communication des réseaux.

  3. Le Modèle TCP/IP :


    4. Le modèle TCP/IP est un modèle de référence pour les
    communications de réseau utilisé dans la conception
    et la mise en œuvre de réseaux informatiques modernes,
    comme les réseaux Internet et les réseaux sans fil.
    Le modèle TCP/IP est divisé en quatre couches principales
    (la couche liaison de données, la couche réseau,
    la couche transport et la couche application).
    Il est le modèle de référence pour les protocoles
    de communication de réseau les plus largement utilisés
    , tels que le protocole Internet (IP) et le protocole de contrôle de transmission (TCP).

  4. L’adressage :


    5. Les communications dans un réseau obéissent à des règles
    d’adressage, d’architecture, et de protocoles pour assurer
    l'échange de données.

    C’est quoi un adressage IP ?

    L'adressage IP est le numéro qui identifie chaque
    ordinateur dans les réseaux, il se décompose a
    deux modes d'adressage IPV4 et IPN6.

  5. L’adresse IPV4:


    6. L’adresse IPV4 se compose d'une partie réseau et
    d'une autre partie hôtes. L'adressage IPv4 est effectué selon quatre
    types d'adresses :

    1. L'adresse réseau :


      2. tous les bits de la partie hôtes sont à zéro,
      permettant à toutes les machines d'un réseau de
      partager la même partie réseau.

    2. L'adresse de diffusion :


      3. tous les bits de la partie hôtes sont à un,
      permettant à un ordinateur d'envoyer des paquets
      à tous les hôtes du réseau sans avoir besoin d'envoyer
      un paquet pour chaque ordinateur de manière unique.

    3. L'adresse hôte :


      4. les valeurs sont situées entre l'adresse réseau et
      l'adresse de diffusion, permettant d'attribuer des
      adresses aux machines situées sur le réseau.

    4. Le masque de réseau et de sous-réseaux :


      5. tous les bits de la partie réseau sont à un, indiquant
      le réseau auquel appartient l'hôte.

Chapitre 5: Système de Gestion de version

  • Définition :


    • L’application Git est un logiciel de versioning où l’utilisateur peut
    stocker les modifications maintenues à son projet et les suivre au
    fil du temps. Ces logiciels nous permettent de travailler en équipe
    d’effectuer des opérations de sorte plus robuste et nous fournirent
    des fonctionnalités de plus. Ils existent deux modèles des logiciels de versioning :
    • modèle centralisé : où le code source est centralisé.
    • modèle distribué : où le code source est hébergé sur un
      serveur distant. Ce modèle est plus simple et flexible au
      travail, et aussi plus sécurisé par rapport au premier.

  • GitHub :


    • c’est un service permettant d’héberger des dépôts
    Git. Notons que n’importe qui peut accéder au code de ces dépôts

  • Installation :


    • la façon la plus adéquate est de télécharger
    la dernière version du site http://git-scm.com/downloads .

  • paramétrage :


    • les commandes utilisées sont :
    • git config : permet de voir et modifier les variables
      de configuration qui contrôlent tous les aspects de
      l’apparence de Git.
    • --global : indiquer à Git que le nom d’utilisateur
      et l’adresse mail renseignés doivent être utilisés globalement.
    • Création de dépôt : Il existe deux façons de
      créer un dépôt Git :
    • Importation d’après un répertoire déjà
      existant.


  • La manière de pensé de Git par rapport aux données
    est différente, ils les considèrent comme étant un flux instantané.
  • La commande « git statuts » est extrêmement nécessaire
    qui s’utilise pour visualiser l’état des répertoires.
  • Parmi ces avantages informer le suivi des versions des
    fichiers et leurs validations l’étape suivante consiste à
    indexer les fichiers pour cela on utilise « git add » et
    cela à travers lui passer le nom du fichier ou du répertoire.
  • On a quelques caractères ayant une signification
    spéciale permettant la création des schémas cela ce qu’utilise
    les fileglobs.
  • Pour la validation des fichiers l’utilisation de « git commit »
    s’impose cela facilite aux contributeurs des projets de savoir maitriser
    les modifications puis les valider.
  • Pour créer un dépôt Git à partir d'un répertoire existant
    , vous pouvez suivre les étapes suivantes :
    • Ouvrez un terminal ou une ligne de commande dans le
      répertoire que vous souhaitez utiliser comme dépôt Git.
    • Initialisez un nouveau dépôt Git
      en exécutant la commande « git init »
    • Ajoutez les fichiers existants
      à l'index Git en utilisant la commande
      « git add ». Cela ajoutera tous les fichiers et dossiers du
      répertoire actuel à l'index Git.
      • Effectuez un premier commit pour enregistrer
      les modifications : « git commit -m "Initial commit" ».
      Cette commande enregistre l'état actuel de tous les fichiers
      dans le dépôt Git avec le message "Initial commit".
  • À partir de là, vous pouvez continuer à travailler sur
    le projet en effectuant des commits et en créant des
    branches pour développer de nouvelles fonctionnalités
    sans impacter le projet de base.
    • Pour ajouter ou modifier des fichiers dans
      un projet et actualiser notre dépôt Git, vous pouvez
      suivre les étapes suivantes :
    • Ajouter les fichiers avec « git add » .
    • Modifier les fichiers existants.
    • Crée un commit avec « git commit ».
    • Pousser les changements avec « git push ».
  • Pour annuler les modifications, on utilise la
    commande « git checkout ». Cette commande
    vous permet de restaurer un fichier modifié à sa
    dernière version enregistrée dans l’historique de Git.

  • Dans Git, une branche est simplement un pointeur vers un
    commit spécifique. Contrairement à d'autres systèmes, Git ne
    crée pas une copie complète du répertoire de travail lors
    de la création d'une nouvelle branche, ce qui facilite la
    création et la fusion de branches. La branche par défaut
    dans Git est appelée "master" mais elle n'est pas
    différente des autres branches. En somme, Git utilise
    des pointeurs pour suivre l'historique du projet et
    créer de nouvelles branches ne nécessite pas la duplication du répertoire de travail.
